# AI App Workspace

这个仓库用于管理一组围绕项目拉取、提示词生成、执行流程记录和结果分析的工作流文件。

它的核心目标不是承载某一个业务系统源码，而是为 session 式项目实验提供统一的组织方式：

- 按项目拉取代码并归档到 session 目录
- 为不同项目生成可复用的 prompt
- 记录模型执行过程和多轮对话结果
- 基于 prompt、代码和结果文件做完成度分析
- 沉淀技能文件、流程文档和工作约定

## 目录结构

```text
apps/
├── .github/              # 自定义 agents 和 instructions
├── flow/                 # 执行流程文档
├── prompts/              # 各 session 的提示词记录
├── session-4/            # session-4 项目代码、结果和模型执行记录
└── skills/               # 任务技能文件，例如拉取项目、分析结果
```

## 主要目录说明

### .github/

存放工作区级别的 agent 和 instruction 配置，用来约束 PromptArchitect、分析流程等行为。

### flow/

存放可复用的操作流程文档，帮助把“拉项目、执行 prompt、记录结果、分析完成度”这一整套动作固定下来。

### prompts/

保存每个 session 下的项目 prompt。通常一个项目只保留当前正在测试的那条 prompt，方便后续和模型执行结果做对比。

### session-4/

存放 session-4 相关的实际项目目录、结果文件和模型对话记录。这里既包含拉下来的项目代码，也包含执行过程中的结果沉淀。

### skills/

存放技能文件，定义具体任务该怎么执行。当前工作区主要包括：

- 项目拉取与 prompt 生成
- 结果分析与评价输出

## 使用方式

一个典型流程通常是这样：

1. 根据项目 ID 拉取代码到对应 session 目录。
2. 为项目生成或筛选本次要测试的 prompt。
3. 用指定模型执行 prompt，并保存 session id 与完整对话。
4. 结合 prompt、代码和结果文件做完成度分析。
5. 将评价结果输出到对应的 result 目录。

## 仓库定位

这是一个用于 AI 项目实验和工作流管理的辅助仓库，重点在于：

- 管理流程
- 组织材料
- 沉淀结果
- 复用技能

不以运行单一业务应用为目标，而是为多项目、多轮模型测试提供统一的管理入口。
