## 测试工程化思路

### 1. 测试分层策略

#### 单元测试 (Unit Tests)
**负责路径**：单个方法的逻辑正确性，不依赖外部系统
**测试范围**：
- `SeckillService` 中的各个方法
- 边界条件验证
- 异常场景验证

**具体测试用例**：
1. **创建活动**：
   - 正常创建活动
   - 秒杀价不低于原价（应抛出异常）
   - 结束时间不晚于开始时间（应抛出异常）
   - 重复创建活动（应抛出异常）

2. **执行秒杀**：
   - 正常秒杀成功
   - 活动不存在（应抛出异常）
   - 活动未开始（应抛出异常）
   - 活动已结束（应抛出异常）
   - 活动状态异常（应抛出异常）
   - 库存不足（应抛出异常）
   - 重复秒杀（应抛出异常）

3. **查询功能**：
   - 查询活动列表
   - 查询活动详情
   - 查询用户订单

#### 集成测试 (Integration Tests)
**负责路径**：端到端的业务流程，依赖数据库等外部系统
**测试范围**：
- 完整的业务流程
- 数据库事务正确性
- 并发场景下的行为验证

**具体测试用例**：
1. **完整业务流程**：
   - 创建活动 → 查询活动 → 执行秒杀 → 查询订单
   - 验证数据一致性

2. **并发场景集成测试**：
   - 多线程并发创建活动（验证唯一约束）
   - 多线程并发秒杀（验证不超卖）
   - 同一用户并发秒杀（验证幂等性）

3. **边界场景集成测试**：
   - 库存为0时的秒杀
   - 活动开始前的秒杀
   - 活动结束后的秒杀

#### 并发测试 (Concurrency Tests)
**负责路径**：高并发场景下的系统行为，验证防超卖逻辑的有效性
**测试范围**：
- 极端并发场景
- 长时间压测
- 不同并发模式

**具体测试用例**：
1. **极端并发场景**：
   - 1000用户抢1件商品
   - 10000用户抢100件商品
   - 验证成功订单数 ≤ 库存数

2. **混合并发场景**：
   - 同时创建活动和执行秒杀
   - 同时查询和修改数据

3. **幂等性验证**：
   - 同一用户100次并发请求
   - 验证只有1次成功

### 2. 验证防超卖逻辑的有效性

#### 核心验证点
1. **成功订单数 ≤ 库存数**：这是最基本的验证
2. **库存扣减一致性**：
   - 初始库存 - 剩余库存 = 成功订单数
   - 验证数据库中的库存值
3. **订单唯一性**：
   - 同一用户不能有多个订单
   - 订单号唯一

#### 验证方法
1. **统计验证**：
   - 统计成功请求数
   - 统计失败请求数（库存不足）
   - 验证成功数 ≤ 库存数

2. **数据库验证**：
   - 查询活动的剩余库存
   - 查询订单表中的订单数
   - 验证：初始库存 - 剩余库存 = 订单数

3. **极端场景验证**：
   - 库存为1，并发1000请求
   - 验证只有1个成功订单
   - 验证库存为0

4. **长时间压测**：
   - 持续压测一段时间
   - 验证系统稳定性
   - 验证数据一致性

##### 集成测试示例
```java
@SpringBootTest
@Transactional
class SeckillConcurrencyTest {

    @Autowired
    private SeckillService seckillService;

    @Autowired
    private SeckillActivityRepository activityRepository;

    @Autowired
    private SeckillOrderRepository orderRepository;

    @Test
    void testConcurrentSeckill_NoOversell() throws InterruptedException {
        // 准备测试数据
        int stock = 10;
        int concurrentUsers = 100;
        
        // 创建活动
        CreateActivityRequest request = CreateActivityRequest.builder()
                .productName("并发测试商品")
                .originalPrice(new BigDecimal("100.00"))
                .seckillPrice(new BigDecimal("10.00"))
                .totalStock(stock)
                .startTime(LocalDateTime.now().minusHours(1))
                .endTime(LocalDateTime.now().plusHours(1))
                .build();
        ActivityVO activity = seckillService.createActivity(request);
        Long activityId = activity.getId();

        // 并发执行秒杀
        ExecutorService executor = Executors.newFixedThreadPool(20);
        CountDownLatch startLatch = new CountDownLatch(1);
        CountDownLatch endLatch = new CountDownLatch(concurrentUsers);
        
        AtomicInteger successCount = new AtomicInteger(0);
        AtomicInteger failCount = new AtomicInteger(0);

        for (int i = 0; i < concurrentUsers; i++) {
            final Long userId = 1000L + i;
            executor.submit(() -> {
                try {
                    startLatch.await(); // 等待所有线程准备就绪
                    seckillService.executeSeckill(activityId, userId);
                    successCount.incrementAndGet();
                } catch (BizException e) {
                    failCount.incrementAndGet();
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    endLatch.countDown();
                }
            });
        }

        // 开始并发执行
        startLatch.countDown();
        endLatch.await();
        executor.shutdown();

        // 验证结果
        SeckillActivity updatedActivity = activityRepository.findById(activityId).orElseThrow();
        long orderCount = orderRepository.count();
        
        // 验证不超卖
        assertTrue(successCount.get() <= stock, "成功订单数不能超过库存数");
        // 验证库存一致性
        assertEquals(stock - successCount.get(), updatedActivity.getAvailableStock().intValue(), "库存扣减不一致");
        // 验证订单数
        assertEquals(successCount.get(), orderCount, "订单数与成功数不一致");
    }
}
```
