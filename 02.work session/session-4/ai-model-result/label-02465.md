## 多环境配置梳理完成

我已经成功为你的账户余额系统完成了多环境配置分层梳理。以下是配置方案的详细说明：

### 后端配置（Spring Boot Profile）

**文件结构：**
- [application.yml](file:///d:/charles/program/ai/apps/session-4/label-02465/backend/src/main/resources/application.yml) - 公共配置（应用名、Jackson、MyBatis Plus、Sa-Token、日志等）
- [application-local.yml](file:///d:/charles/program/ai/apps/session-4/label-02465/backend/src/main/resources/application-local.yml) - 本地开发环境（H2 内存数据库，启用 H2 Console）
- [application-dev.yml](file:///d:/charles/program/ai/apps/session-4/label-02465/backend/src/main/resources/application-dev.yml) - 集成自测环境（MySQL，容器内连接）
- [application-prod.yml](file:///d:/charles/program/ai/apps/session-4/label-02465/backend/src/main/resources/application-prod.yml) - 生产环境（MySQL，支持环境变量覆盖，日志级别调整为 WARN）

**使用方式：**
- 本地开发：默认使用 `local` Profile（H2 数据库）
- 容器内：通过 `SPRING_PROFILES_ACTIVE=dev` 或 `SPRING_PROFILES_ACTIVE=prod` 指定
- IDE 运行：在 VM options 中添加 `-Dspring.profiles.active=local`

### 前端配置（Vite 环境变量）

**文件结构：**
- [.env.local](file:///d:/charles/program/ai/apps/session-4/label-02465/frontend-admin/.env.local) - 本地开发环境（配置代理目标地址）
- [.env.dev](file:///d:/charles/program/ai/apps/session-4/label-02465/frontend-admin/.env.dev) - 集成自测环境
- [.env.prod](file:///d:/charles/program/ai/apps/session-4/label-02465/frontend-admin/.env.prod) - 生产环境
- [vite.config.js](file:///d:/charles/program/ai/apps/session-4/label-02465/frontend-admin/vite.config.js) - 已更新为支持环境变量配置代理

**使用方式：**
- `npm run dev` 或 `npm run dev:local` - 本地开发模式
- `npm run dev:dev` - 使用 dev 环境配置
- `npm run build` 或 `npm run build:prod` - 构建生产版本
- `npm run build:dev` - 构建 dev 版本
- `npm run build:local` - 构建本地版本

### Docker Compose 配置

**文件结构：**
- [docker-compose.yml](file:///d:/charles/program/ai/apps/session-4/label-02465/docker-compose.yml) - 基础配置（dev 环境，使用 `dev` Profile）
- [docker-compose.prod.yml](file:///d:/charles/program/ai/apps/session-4/label-02465/docker-compose.prod.yml) - 生产环境配置（支持环境变量覆盖数据库密码等敏感信息）

**使用方式：**
- 集成自测环境：`docker-compose up -d`
- 生产环境：`docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d`
- 生产环境可通过环境变量覆盖配置：
  ```bash
  DB_ROOT_PASSWORD=your_secure_password DB_PASSWORD=your_secure_password docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d
  ```

| 集成自测 | dev | MySQL (容器) | `npm run build:dev` | `docker-compose up -d` |
| 生产 | prod | MySQL (支持环境变量) | `npm run build:prod` | `docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d` |
