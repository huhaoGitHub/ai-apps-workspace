### 二、核心改造点

#### 1. 格式标准化
```
控制台格式（带颜色）：
2026-04-04 10:30:00.123 INFO 12345 --- [http-nio-8080-exec-1] [abc123def456] c.t.c.OperationLogAspect : [OPERATION] [abc123def456] 开始执行...

文件格式：
2026-04-04 10:30:00.123 [http-nio-8080-exec-1] INFO [abc123def456] com.taskman.common.OperationLogAspect - [OPERATION] [abc123def456] 开始执行...
```

**统一字段**：`时间戳` + `线程名` + `日志级别` + `TraceID` + `类名` + `消息`

#### 2. 分级输出
| 日志文件 | 级别 | 说明 |
|----------|------|------|
| `task-manager-info.log` | INFO | 业务操作日志，按天滚动压缩 |
| `task-manager-error.log` | ERROR+ | 错误日志，独立文件便于排查 |

**滚动策略**：单文件 100MB，保留 30 天，总大小 3GB

#### 3. 异常堆栈追踪
- `BusinessException`：WARN 级别，记录错误码和消息（无堆栈，业务预期异常）
- `IllegalArgumentException`：WARN 级别，记录完整堆栈
- `Exception`（兜底）：ERROR 级别，记录完整堆栈 + TraceID 返回给用户

#### 4. 调用链追踪（TraceID）
- 自动生成 16 位 TraceID（UUID 截取）
- 支持从 HTTP Header `X-Trace-Id` 透传（微服务场景）
- MDC 上下文传递，所有日志自动携带
- 系统异常时 TraceID 会返回给前端，便于用户反馈问题

#### 5. 环境分离

| 配置项 | 开发环境 (dev) | 生产环境 (prod) |
|--------|----------------|-----------------|
| 控制台输出 | ✅ 开启 | ❌ 关闭 |
| 日志级别 | DEBUG | INFO |
| SQL 日志 | DEBUG | WARN |
| 日志路径 | `./logs` | `/var/log/task-manager` |
| 保留天数 | 7 天 | 30 天 |

**切换方式**：
```bash
# 环境变量
SPRING_PROFILES_ACTIVE=prod

# 或 JVM 参数
java -jar app.jar --spring.profiles.active=prod
```

### 四、使用建议

1. **排查问题**：用 TraceID 搜索 `task-manager-error.log`，可快速定位完整调用链
2. **日志分析**：INFO 和 ERROR 分离，ELK 采集时可分别处理
3. **性能优化**：生产环境使用 AsyncAppender，不阻塞业务线程
4. **透传 TraceID**：如果有网关/其他服务，在 HTTP Header 中传递 `X-Trace-Id` 即可串联调用链
