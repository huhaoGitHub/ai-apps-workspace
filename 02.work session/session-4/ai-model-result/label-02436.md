### 四、查问题时的优势

1. **按 traceId 串联** - 同一个请求的操作日志、异常日志、SQL 日志都有相同的 traceId，可以快速关联
2. **独立文件** - 想看 SQL 直接查 `sql.log`，想看异常直接查 `exception.log`，不用在混杂的日志中过滤
3. **JSON 格式** - 操作日志是 JSON，方便后续接入 ELK 等日志分析系统
4. **完整上下文** - 每条日志都包含请求 URI、方法、客户端 IP 等信息
